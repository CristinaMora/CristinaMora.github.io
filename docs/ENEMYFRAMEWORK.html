Enemy Behaviour Framework

Bachelor Thesis - Universidad Complutense de Madrid

Project Summary

The Enemy Behaviour Framework is a modular toolkit designed for Unity that enables non-programmers to create complex enemy behaviours in 2D platform games through a visual component system. The aim was to separate logic from design, allowing game designers to work independently from programmers while preserving expressive capabilities.

Motivation

As 2D games evolve, enemy behaviour design becomes more demanding. This project addresses the need for accessible tools that reduce technical barriers and streamline the creation of enemy AI for designers with limited coding skills.

Main Components

  • Sensors: Detect conditions in the environment (distance, collision, time, etc.)
  • Actuators: Define enemy actions (movement, attack, spawning)
  • Damage Module: Manages how enemies deal and receive damage
  • Finite State Machine (FSM): Organizes behaviours and transitions

User Interface

A user-friendly interface was implemented in Unity, offering designers a drag-and-drop experience using prefabs and inspector-based configuration. Each behaviour component is reusable and independent, promoting modularity.

Evaluation

The tool was tested with users unfamiliar with programming. Results showed that users were able to design functional enemies with minimal guidance, demonstrating the accessibility and effectiveness of the framework.
