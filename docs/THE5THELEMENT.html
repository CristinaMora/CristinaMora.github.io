THE 5TH ELEMENT

Game Overview

The Fifth Element is a hybrid game that combines classic beat 'em up combat with top-down exploration mechanics. The story begins with the player receiving a letter from a deceased sibling, prompting a quest to reunite the remaining four siblings scattered across five elemental kingdoms. Throughout the journey, the player must overcome enemies representing different elemental types and recruit each sibling into their team. The game allows seamless switching between siblings during combat to exploit elemental weaknesses and develop team-based strategies.

Technology Stack

  • Engine: SDL
  • Language: C++
  • Tools: Custom built systems without external libraries

Core Architecture

The project initially used a classic inheritance-based architecture with a generic Entity class. As the scope expanded, the team transitioned to a component-based architecture to support greater flexibility and maintainability. Game states are managed via a singleton GameStateMachine, responsible for activating and disposing of each game state cleanly. The architecture supports two primary modes: a top-down overworld mode for exploration, and a side-view beat 'em up mode for combat sequences.

Gameplay Features

  • Dual gameplay: Top-Down Exploration and Beat 'Em Up Combat
  • Element-based combat: enemies and characters aligned with elemental themes
  • Character switching: real-time sibling swap during battles
  • Narrative-driven progression through five unique kingdoms
  • Recruitment mechanic: each sibling joins as the story advances

Development Context

This project stood out for the freedom it offered in both gameplay and narrative design. Our group consisted of eleven members, which required structured coordination. I led weekly meetings and worked to maintain a relaxed, productive environment. Being involved in nearly every development area gave me a holistic understanding of game creation, from technical systems to team dynamics.
